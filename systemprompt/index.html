Prompts de Sistema vs. Prompts de Usuario

Cómo funciona la jerarquía de instrucciones en la IA, dónde se configura cada nivel y cómo aplicarlo en herramientas personalizadas como GPTs, GEMs y Proyectos de Claude.

Idea central: No toda instrucción que le das a la IA tiene el mismo peso. El prompt de sistema es la constitución — define las reglas que nunca cambian. El prompt de usuario es la petición dentro de esas reglas. Entender esta jerarquía es la base del prompting profesional.

Los dos niveles de comunicación con la IA
Nivel 1 — Mayor autoridad
Prompt de Sistema
Las instrucciones maestras. Se configura antes de que comience la interacción y define las reglas de juego para toda la sesión.
→ Define el rol y la personalidad de la IA
→ Establece restricciones y límites éticos
→ Persiste durante toda la conversación
→ El usuario no puede anularlo
Nivel 2 — Subordinado al sistema
Prompt de Usuario
El mensaje directo del humano en la interfaz. Representa la necesidad inmediata dentro del marco definido por el sistema.
→ Activa una tarea específica
→ Cambia en cada turno de conversación
→ Opera dentro de las reglas del sistema
→ Define el resultado que se desea

Conceptos clave
Jerarquía de instrucciones
El sistema siempre tiene más autoridad que el usuario. Si el sistema dice "no ofrezcas descuentos", ningún usuario puede obligar a la IA a ofrecerlos.
Persistencia vs. efímero
El sistema dura toda la sesión. El usuario cambia en cada mensaje. Esta distinción determina qué debe ir en cada nivel al diseñar un asistente.
Separación de roles
El creador del asistente escribe el sistema. El usuario final escribe sus prompts. En herramientas personalizadas, estos dos roles raramente son la misma persona.
Marco de comportamiento
El sistema no solo restringe — también amplía. Puede dar al modelo acceso a herramientas, bases de conocimiento y comportamientos que sin el sistema no tendría.
La jerarquía en una imagen
Sistema ↓ define el marco Nivel de mayor autoridad. Establece las reglas, el rol, el tono y las restricciones. Ninguna instrucción del usuario puede anularlo.
Usuario ↓ opera dentro Opera dentro del marco definido por el sistema. Puede pedir lo que quiera, pero la IA solo responderá si la petición es compatible con las reglas del sistema.
Respuesta ← resultado La intersección de ambos niveles. La IA responde a la petición del usuario sin violar las instrucciones del sistema.
Analogías para entenderlo
Prompt de sistema
El manual de identidad corporativa de una empresa. Define cómo habla la marca, qué puede y no puede prometer, y cuál es su propósito. Existe antes de que llegue ningún cliente.
vs.
Prompt de usuario
El pedido de un cliente específico. Puede pedir lo que quiera, pero el empleado (la IA) solo hará lo que está dentro del manual de la empresa.
Prompt de sistema
La constitución de un país. Es la norma suprema que establece los principios fundamentales. Las leyes ordinarias (los usuarios) no pueden contradecirla.
vs.
Prompt de usuario
Una ley ordinaria o decreto. Puede crear reglas específicas, pero siempre dentro del marco constitucional. Si la contradice, no tiene efecto.
Prompt de sistema
Las instrucciones de un empleado nuevo en su primer día: rol, valores de la empresa, qué puede y no puede hacer, cómo hablar con los clientes.
vs.
Prompt de usuario
La consulta de un cliente que llega a ese empleado. El empleado responde según lo que aprendió el primer día, no según lo que el cliente le pida que haga.
Dónde se configura el prompt de sistema en cada plataforma

Las principales plataformas de IA permiten crear asistentes personalizados. En todos los casos, el prompt de sistema es el campo donde configuras las instrucciones permanentes. El nombre del campo varía según la plataforma.

Gemini — GEMs
gemini.google.com → Mis Gems → Editar
Nombre del campo en la interfaz
Instrucciones — es el área de texto principal al editar un GEM
Qué escribir aquí
El rol del GEM, su especialización, el tono de respuesta, qué puede y no puede hacer, y las fuentes que debe priorizar. También puedes cargar archivos en la sección Conocimiento para que el GEM los use como base de referencia.
Ejemplo de instrucción de sistema
Eres un editor de estilo especializado en comunicación corporativa. Siempre corriges usando las normas de la RAE. Nunca alteres el significado del texto original. Si encuentras laísmo o dequeísmo, señálalo con una nota explicativa antes de la corrección.
Diferencia clave: en Gems, el usuario final solo ve la interfaz de chat — nunca ve las instrucciones que escribiste. El sistema opera de forma invisible.
ChatGPT — GPTs personalizados
chatgpt.com → Mis GPTs → Configurar
Nombre del campo en la interfaz
Instructions — en la pestaña "Configurar" del panel de edición del GPT
Qué escribir aquí
El rol del GPT, su comportamiento, qué herramientas puede activar (búsqueda web, generación de imágenes, análisis de datos) y bajo qué condiciones. También defines las restricciones de seguridad y el alcance de las respuestas.
Ventaja adicional de GPTs
Puedes definir en el sistema qué herramientas integradas tiene disponibles el GPT: si puede buscar en internet, generar imágenes con DALL-E o ejecutar código Python. El usuario no puede activar herramientas que el sistema no haya habilitado.
Buena práctica: incluye en las instrucciones las "reglas de oro" del negocio — frases como "nunca ofrezcas reembolsos, solo soluciones técnicas" o "escala al humano si no puedes resolver en 3 intentos".
Claude — Proyectos
claude.ai → Proyectos → Instrucciones del proyecto
Nombre del campo en la interfaz
Instrucciones del proyecto — en la configuración del Proyecto, se aplican a todos los chats dentro de ese proyecto
Qué escribir aquí
El comportamiento base que Claude debe mantener en todas las conversaciones del proyecto: rol, tono, restricciones y fuentes de verdad. Puedes también cargar archivos técnicos o manuales para que Claude responda basándose exclusivamente en esa documentación.
Ventaja adicional de Proyectos
Las instrucciones del proyecto se aplican automáticamente a todos los chats nuevos dentro del proyecto, sin necesidad de repetirlas. Ideal para equipos que comparten un mismo asistente especializado.
Caso de uso ideal: carga manuales internos o normativas como archivos del proyecto y escribe en las instrucciones: "Responde únicamente basándote en los documentos cargados. Si la información no está en ellos, indícalo explícitamente."
El ejemplo completo — soporte técnico de software

Este ejemplo muestra cómo los tres niveles interactúan: el sistema define las reglas, el usuario hace una petición que incluye algo que el sistema prohíbe, y la IA responde respetando las reglas del sistema sin importar lo que pida el usuario.

Asistente de Soporte Técnico — cómo actúan los tres niveles
Sistema Eres un técnico de nivel 2. Tu tono es paciente...
Eres un técnico de soporte de nivel 2. Tu tono es paciente y estructurado. Regla de oro: no ofrezcas reembolsos bajo ninguna circunstancia, solo soluciones técnicas. Si el problema persiste tras 3 intentos de solución, escala el caso al departamento humano.
Lo que hace este sistema: define el rol (técnico L2), el tono (paciente), una restricción crítica (no reembolsos) y un protocolo de escalada. Todo esto opera en segundo plano — el usuario no lo ve.
Usuario Mi aplicación se cierra sola al abrir un PDF. ¡Exijo que me devuelvan mi dinero!
Mi aplicación se cierra sola al abrir un archivo PDF. ¡Exijo que me devuelvan mi dinero!
Lo que hace este prompt: contiene dos cosas — un problema técnico legítimo (la app se cierra) y una petición que el sistema prohíbe (el reembolso). La IA debe atender lo primero e ignorar lo segundo.
Resultado de la IA Siento mucho los inconvenientes. Como técnico, mi prioridad es resolver...
Siento mucho los inconvenientes que está experimentando. Como técnico, mi prioridad es resolver el error de apertura del PDF para que pueda usar la aplicación con normalidad. Probemos primero borrando la caché: vaya a Configuración → Almacenamiento → Borrar caché y reinicie la aplicación. ¿El problema persiste después de ese paso?
Lo que demuestra este resultado: la IA reconoció la frustración del usuario (tono empático), atendió el problema técnico con un paso concreto, e ignoró completamente la exigencia del reembolso — exactamente como el sistema indicó. El usuario no puede "romper" esa regla.
Qué aprender de este ejemplo
El sistema es invisible pero omnipresente
El usuario nunca vio las instrucciones del sistema, pero cada respuesta de la IA estuvo moldeada por ellas. Las "reglas de oro" operan en segundo plano siempre.
El tono lo define el sistema, no el usuario
Aunque el usuario escribió en mayúsculas con signo de exclamación, la IA respondió con calma. El tono "paciente y estructurado" del sistema prevalece sobre el tono del usuario.
El sistema puede incluir protocolos completos
La instrucción de escalar al humano tras 3 intentos es un protocolo operativo completo escrito en el sistema. El usuario no puede saltárselo, y la IA lo seguirá automáticamente.
